This Frontend structure is to make it easy to switch to 
any CSS FRAMEWORK at anytime.

The main templates folder at the django base directory 
is used to keep the imports from CSS Frameworks

Remember to tell django to load the template files 
from the base directory too.
This is done by the including the following code  in the settings.py


TEMPLATES = [
    {
        'BACKEND': 'django.template.backends.django.DjangoTemplates',
        'DIRS': [os.path.join(BASE_DIR, 'templates')], # This tells django to look in the base directory/templates folder too.
        'APP_DIRS': True, # This tells django to look in the app directories too.
        'OPTIONS': {
            'context_processors': [
                'django.template.context_processors.debug',
                'django.template.context_processors.request',
                'django.contrib.auth.context_processors.auth',
                'django.contrib.messages.context_processors.messages',

                'django.template.context_processors.media',
                'access_control_and_custom_settings.user_access_control_lists.user_access_lists',
            ],
        },
    },
]



SUMMARY OF THE ARCHITECTURE: (TEMPLATE COMPONENTS TREE)
===============================

base.html/
    |-- {% includes %}- bootstrap_CDN.html
            |-- # This imports all the bootstrap5 js, css, etc.
            # this bootstrap_CDN.html may not even be necessary as we can import the js and css files 
            # directly - and besides the heavy js files should be imported at the bottom of the page
            # as a convention for best practice and better performance.

    |-- side_nav.html
            |-- {% includes %}- 'all_partials.html'
                                      |-- {% includes %}-- (all the partials for the nav-menus) 

    |-- footer.html'

===============================



NOW THE DETAILED ARCHITECTURE IS SIMPLE:

The <css_framework_name>-components folder will contain the css imports for a specified CSS FRAMEWORK
    e.g. tailwind-components, bootstrap5-components, etc

base.html will import the css static files and js files from 
django static folder:
Remember, django will not allow you to import any js files outside 
the static folder.

<script src="/<BASE_DIR>/static_files/bootstrap_5_1_3_dist/js/bootstrap.bundle.min.js"></script>
# where <BASE_DIR> represents your actual base directory

<link href="/<BASE_DIR>/static_files/bootstrap_5_1_3_dist/css/bootstrap.min.css" rel="stylesheet">


Also, the nav_bar and menus will be in the folder <css_framework_name>-components/containers and 
    <css_framework>-components/navigation_menus
    respectively.

Hence base.html will import them like so:

        <!-- the 0_framework_name-CDN is just a file for importing the js scripts-->
        e.g. {% include 'tailwind-components/0_tailwind-CDN.html' %}

        <!-- Include the Navbar or SideBar -->
        e.g. {% include 'tailwind-components/containers/navbar-tailwind.html' %}



LASTLY, if you are using HTMX then remember to include the ff script to allow for csrf_token to work


<script src="/<BASE_DIR>/static_files/js/htmx.js"></script>

<script>
        document.body.addEventListener('htmx:configRequest', (event) => {
            event.detail.headers['X-CSRFToken'] = '{{ csrf_token }}'; //insert csrf token when performing AJAX request 
        })
</script>


SO THEREFORE, every css framework MUST be organized to have the following folder structure
Hence once we swap it into our project - we effectively have a new Frontend Layout.


Folder Structure explained below:

base.html/
    |-- {% includes %}- bootstrap5-components/0bootstrap-CDN.html
                                #-- This imports all the bootstrap5 js, css, etc.

    |-- {% includes %}- 'bootstrap5-components/containers/side_nav.html
                            |-- {% includes %}- 'bootstrap5-components/navigation_menus/all_partials.html'
                            |-- {% includes %}- 'bootstrap5-components/navigation_menus/login_logout_links.html'

    |-- {% includes %}- 'bootstrap5-components/containers/_footer.html'


    # Also, any page that has pagination can include the pagination links from the main template
    # like so: 
    # {% includes %}- 'pagination_partial/_pagination_links_partial.html'
